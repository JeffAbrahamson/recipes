# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A LaTeX recipe book compiled into device-specific PDFs (laptop, iPad, Nexus 5, Nexus 9). Each device format sets different page dimensions; all share the same content via `recipes.tex`.

## Build

```bash
make          # builds all four PDFs (nexus9, nexus5, ipad, laptop)
make indices  # regenerates index.tex files in each chapter directory (run before building)
make clean    # removes all build artifacts
```

Building a single target:
```bash
make recipes-laptop.pdf
```

The build pipeline for each PDF: `xelatex` → `makeindex` → `bibtex` → `xelatex` × 2. Output PDFs are moved to `pdf/`.

**Requires**: `xelatex`, `makeindex`, `bibtex`, and the Noto CJK fonts (for Japanese support).

## Adding a recipe

1. Create a `.tex` file in the appropriate chapter directory (`breakfasts/`, `salads/`, `misc/`, `soup/`, `sides/`, `mains/`, `desserts/`).
2. Use `template.tex` as the starting point.
3. Run `make indices` to regenerate the `index.tex` files (the `make-dir` script handles this automatically).

## LaTeX macros (`recipe-macros.tex`)

Key macros for writing recipes:

| Macro | Purpose |
|---|---|
| `\dish{Name}` | Recipe title; adds to TOC and index |
| `\altdish{Name}` | Alternative name; adds to index only |
| `\subdish{Name}` | Sub-recipe heading |
| `\serves{N}` / `\pour{N}` | Serving size (English/French) |
| `\makes{N}` / `\fait{N}` | Yield (English/French) |
| `\prep{text}` | Prep time / notes in header |
| `\source{key}` | Cite from `recipes.bib` |
| `\sourceD{key}{note}` | Cite with page/annotation |
| `\sourcep{text}` | Free-form source line |
| `\stage{text}` | Bold section heading within recipe |
| `\notes` | Starts a "Notes:" section |

Ingredient macros (used inside `ingredients` environment):
- `\ingr{qty}{unit}{description}` — standard ingredient
- `\ingrS{qty}{unit}{description}` — ingredient with extra space after (use to separate groups)
- `\ingrr{qty1}{unit1}{qty2/unit2 ...}{description}` — two-column quantities (for scaled recipes, use `ingredientss` environment)

Temperature / fraction helpers: `\FC{F}{C}`, `\CF{C}{F}`, `\fracH` (½), `\fracQ` (¼), `\fracQQ` (¾), `\fracT` (⅓), `\fracTT` (⅔), `\tsp`, `\tbsp`.

French helpers: `\cs` (c. à soupe), `\ccf` (c. à café), `\accord{wine}`.

## Document structure

```
recipes.tex          # master include file (chapters, bibliography, index)
book-format.tex      # page geometry, headers, hyperlinks
recipe-macros.tex    # all recipe-specific LaTeX macros
recipes-laptop.tex   # sets page dimensions, inputs recipes.tex
recipes-ipad.tex     # ipad portrait dimensions
recipes-nexus5.tex   # Nexus 5 dimensions
recipes-nexus9.tex   # Nexus 9 dimensions
recipes.bib          # BibTeX bibliography
make-dir             # shell script: generates index.tex for a chapter directory
template.tex         # blank recipe template
```

Chapter directories: `breakfasts/`, `salads/`, `misc/`, `soup/`, `sides/`, `mains/`, `desserts/`.

Each chapter directory has an `index.tex` generated by `make-dir` that `\input`s all `.tex` files in that directory (excluding `index.tex` itself).

## Importing recipes

**Tone**: Recipe sites and cookbooks write for anxious beginners and explain technique for education — strip both out. Remove reassurances, hand-holding, and filler. Strip the educational layer: a cookbook explains *why* the fold-and-pass develops gluten; a recipe just says to fold and pass. Write as if the reader knows how to cook. Keep a technique note only when the method is genuinely non-obvious or would surprise a competent cook.

**Word economy in steps**: State the action and any non-obvious parameters. Cut everything else. Specific patterns to eliminate:
- Method implied by the verb: "former une boule" needs no "en pressant bien avec les mains"; "reposer dans un torchon humide" needs no separate "emballer" step
- Purpose clauses when obvious: "pour arrêter la cuisson", "pour en extraire le jus" — cut
- Outcome narration: "elle ne sera pas lisse à ce stade", "ce qui donne 8 morceaux" — cut
- "Bien" as intensifier: "bien égoutter" → "égoutter"
- Verbose relative clauses: "de façon à ce que les ingrédients y baignent sans être recouverts" → "juste à hauteur"
- Split actions that belong in one sentence: "Mettre dans une casserole. Ajouter X. Porter à ébullition" → "Porter X à ébullition"
- Fluff words: "filtered water" → "water".

Apply this even when the source is verbose. The goal is the minimum a competent cook needs, not a summary of the source's prose.  Excess text distracts the cook whilst cooking.

**Calibration**: Before finalising a recipe, compare its step density against `misc/ramen.tex`. That recipe is a good style target — short imperative sentences, parameters inline or in parentheses, no subordinate clauses unless essential. If a step looks longer than a comparable step there, cut further.

**Naming**: Use commonly-known names in source language, avoid flowery or verbose naming unless it adds value.

**Language**: Recipes may be in English, French, or German. Keep the original language; do not translate unless explicitly asked.

**Ingredient ordering**: List ingredients in the order they are first used in the recipe. Use `\ingrS` on the last ingredient of a group to add a small visual gap before the next group, marking a shift in the cook's focus (e.g., wet ingredients → dry ingredients, sauté phase → braising liquid). For recipes with clearly distinct components that require their own preparation steps, use separate `\begin{ingredients}...\end{ingredients}` and `\begin{recipe}...\end{recipe}` blocks under `\stage{}` headings, rather than one long ingredient list.

**Titles in CJK scripts**: Recipe titles may be in Japanese or Chinese if that is the original name.
- Japanese: accompany kanji with hiragana or katakana (via `\hiragana{}`) unless the kanji are widely known.
- Chinese: accompany hanzi with pinyin unless the characters are widely known.

added.  If the commit represents changes beyond simple clarification,
that would merit a further message, otherwise "Improved X" is enough.
